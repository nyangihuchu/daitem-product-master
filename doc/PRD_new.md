# 다이템 MVP PRD

## 🎯 핵심 정보
**목적**: 카페24·네이버스토어·쿠팡 등 7개 온라인 몰에서 주문을 받고 → 공급처에 발주하여 → 판매가와 매입가의 마진을 관리하는 공구 유통 운영 전체를 단일 백오피스에서 처리한다.
**사용자**: 자사몰(카페24) + 네이버스토어·쿠팡·지마켓 등 7개 판매 채널을 운영하는 공구 유통 사업 내부 운영 팀 (관리자, 운영자, 조회 전용 역할 구분)

### 판매 핵심 프로세스
```
① 주문 확인       →  ② 공급처 발주       →  ③ 마진 계산
온라인 몰            공급처로 상품           판매가 - 매입가 -
(7개 채널)           발주 처리               마켓 수수료 = 실질 마진

[현재 - MVP]         [현재 - MVP]
직접 몰에서 확인 후   발주서 수동 생성 후
시스템에 수동 입력    공급처에 전달

[향후 - 자동화]      [향후 - 자동화]
각 몰 API 연동으로    주문 수신 시
주문 자동 수집        발주서 자동 생성·발송
```

> **설계 방향**: MVP는 수동 입력으로 시작하되, 향후 API 연동 자동화로 확장할 수 있도록
> 주문 데이터 구조와 발주 처리 로직을 처음부터 자동화 확장을 고려하여 설계한다.

## 🚶 사용자 여정

```
1. 서비스 진입 (미인증)
   ↓ URL 직접 접근 → 로그인 페이지로 리디렉션
   로그인 페이지 ✅ 개발완료
   ↓ 로그인 성공

2. 대시보드 페이지 (메인 허브)
   ↓ 오늘 주문·미처리 발주·이달 매출·이익 현황 확인
   ↓ 재고 부족 상품 알림 카드 → 상품 관리 페이지로 이동
   ↓ D-day 임박 일정 클릭 → 일정·알림 페이지로 이동
   ↓ 사이드바 메뉴 선택
   ├── [상품 관리] → 3단계
   ├── [공급처 관리] → 4단계
   ├── [주문 관리] → 5단계
   ├── [발주 관리] → 6단계
   ├── [수익 분석] → 7단계
   ├── [정산 관리] → 8단계
   └── [일정·알림] → 9단계

3. 상품 관리 페이지
   ↓ 다중 필터로 12만 개 상품 탐색 (페이지네이션)
   ↓ "상품 등록" → 상품 등록 폼 (슬라이드 패널)
   ↓ 저장 완료 → 테이블 반영
   ↓ "엑셀 업로드" → 일괄 등록 처리

4. 공급처 관리 페이지
   ↓ "공급처 등록" → 등록 폼 (슬라이드 패널)
   ↓ 저장 완료 → 공급처 목록 반영

5. 주문 관리 페이지  ← [STEP 1] 온라인 몰에서 주문 확인
   [현재] 각 온라인 몰에서 직접 주문을 확인한 후 시스템에 수동 입력
   [향후] 각 몰 API 연동으로 주문 자동 수집 (입력 불필요)
   ↓ 주문별 상품·수량·채널·판매가 확인
   ↓ 처리할 주문 선택 → "발주 생성" 클릭 → 6단계

6. 발주 관리 페이지  ← [STEP 2] 공급처로 상품 발주
   [현재] 주문 선택 후 발주서 수동 생성 → 공급처에 전달
   [향후] 주문 수신 즉시 발주서 자동 생성·발송
   ↓ 동일 공급처 복수 주문 자동 묶음 처리
   ↓ PDF 출력 또는 이메일로 공급처에 발주서 전달
   ↓ 발주 완료 → 주문 상태 "발주완료"로 자동 전환

7. 수익 분석 페이지  ← [STEP 3] 마진 계산 및 수익 확인
   ↓ 마진 계산식 확인: 판매가(마켓 판매가) - 매입가(공급처 구입가) - 마켓 수수료 = 실질 마진
   ↓ 채널별·기간별 매출 및 마진율 통계 조회

8. 정산 관리 페이지
   ↓ 마켓별 정산 예정일·예상 금액 확인
   ↓ 미정산 잔액 현황 파악

9. 일정·알림 페이지
   ↓ "일정 등록" → 일정 유형 선택 (공급처/마켓/내부/고객 관련)
   ↓ 알림 채널 설정 (이메일/브라우저 푸시)
   ↓ D-day 도달 시 자동 알림 발송 → 담당자 수신
```

### 3. MVP 이후 기능 — 자동화 확장 단계

> MVP에서 수동으로 처리하는 **주문 확인(F009)**과 **발주 처리(F010)**를 자동화하는 것이 다음 단계의 핵심 목표.
> MVP 설계 시 `source_type`, `trigger_type` 필드를 포함해 자동화 전환 시 데이터 구조 변경 없이 확장 가능하도록 준비.

#### 자동화 핵심 (최우선)
- **7개 마켓 주문 자동 수집** — 카페24·네이버스토어·쿠팡·지마켓·옥션·롯데온·11번가 API 연동으로 주문 자동 등록 (F009 수동 입력 대체)
- **발주 자동화** — 주문 수신 즉시 발주서 자동 생성·공급처 발송 (F010 수동 생성 대체)
- **카페24 API 연동** — 상품 정보 양방향 동기화 (`cafe24_product_id` 필드 MVP부터 포함)

#### 운영 고도화
- 스마트택배 API 배송 추적 연동
- 카카오 알림톡·SMS 알림 채널 추가
- 상품 이미지 업로드 및 관리
- 광고비 집행 관리 및 마진 반영
- 반품·불량률 자동 비율 추정

> **설계 원칙**
>
> **자동화 확장 대비 (핵심)**
> - `orders.source_type` 필드(`manual` / `api`)를 MVP부터 포함 → 향후 마켓 API 연동 시 데이터 구조 변경 없이 자동 수집 전환 가능
> - 발주 생성 로직을 서비스 레이어로 분리 → 수동 트리거(버튼 클릭)와 자동 트리거(주문 수신 이벤트) 모두 동일 로직 재사용
> - `purchases.trigger_type` 필드(`manual` / `auto`) 포함 → 자동 발주 전환 후에도 수동 발주 병행 가능
> - API Routes는 RESTful 구조로 설계 → 카페24 웹훅·마켓 API 연동 시 엔드포인트 추가만으로 확장
>
> **데이터 관리**
> - `products.cafe24_product_id` 컬럼을 Phase 1부터 포함 (카페24 API 연동 대비)
> - 매입가·정산 데이터 물리 삭제 금지 → 소프트 딜리트(`deleted_at`)만 허용 (회계 기록)
> - 마켓 수수료율: `market_fees` 테이블 분리로 변동 시 이력 관리
>
> **성능**
> - 상품 목록 페이지: 페이지네이션 + DB 인덱스 최적화 (12만 개 대응)
> - 엑셀 업로드: 청크 단위 분할 처리 + Supabase bulk insert 연계
