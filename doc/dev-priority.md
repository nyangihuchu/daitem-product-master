# 개발 우선순위

> 기준: 의존성 순서 (하위 레이어 먼저) + 핵심 가치 먼저

---

## Phase 1 — 기반 설정

데이터와 인증이 없으면 아무것도 동작하지 않음. 모든 개발의 전제 조건.

| 순서 | 작업 | 관련 기능 |
|------|------|----------|
| 1 | Supabase 프로젝트 생성 및 환경변수 설정 | — |
| 2 | DB 스키마 생성 (`users`, `suppliers`, `products`) | — |
| 3 | Next.js 라우트 그룹 구조 설정 (`(auth)`, `(dashboard)`) | — |
| 4 | 로그인 페이지 구현 | F011 |
| 5 | 회원가입 페이지 구현 | F011 |
| 6 | 인증 미들웨어 설정 (미로그인 시 로그인 리디렉션) | F011 |
| 7 | 대시보드 레이아웃 구현 (사이드바 + 헤더) | — |

---

## Phase 2 — 공급처 관리

상품 등록 시 공급처를 반드시 선택해야 하므로 상품보다 먼저 구현.

| 순서 | 작업 | 관련 기능 |
|------|------|----------|
| 8 | 공급처 목록 페이지 (카드형 UI, 국내/해외 탭 필터) | F002 |
| 9 | 공급처 등록/수정 폼 (슬라이드 패널) | F001 |
| 10 | 공급처 삭제 (확인 다이얼로그) | F001 |

---

## Phase 3 — 상품 관리

서비스 핵심 기능. 마진 자동 계산과 재고부족 자동 전환이 이 단계의 핵심.

| 순서 | 작업 | 관련 기능 |
|------|------|----------|
| 11 | 상품 목록 페이지 (테이블형) | F003, F006 |
| 12 | 상품 검색 및 필터 (SKU·상품명 검색, 공급처·상태·플랫폼 필터) | F006 |
| 13 | 상품 등록/수정 폼 (슬라이드 패널) | F003, F012 |
| 14 | 마진액·마진율 실시간 자동 계산 (폼 내) | F004 |
| 15 | 재고 <= 최소 재고 시 "재고부족" 상태 자동 전환 | F005 |
| 16 | 상품 삭제 (확인 다이얼로그) | F003 |

---

## Phase 4 — 재고 현황

상품 데이터가 있어야 의미 있는 화면. Recharts 바 그래프 구현 포함.

| 순서 | 작업 | 관련 기능 |
|------|------|----------|
| 17 | 재고 현황 페이지 (수평 바 그래프, 재고부족 상단 정렬) | F007 |
| 18 | 바 그래프 색상 구분 (정상·주의·부족) | F007 |
| 19 | 인라인 재고 수량 수정 및 저장 | F008 |
| 20 | 재고 수정 후 재고부족 상태 자동 재계산 | F005 |

---

## Phase 5 — 대시보드

앞선 모든 데이터를 집계하는 마지막 화면. 데이터가 충분히 쌓인 후 구현.

| 순서 | 작업 | 관련 기능 |
|------|------|----------|
| 21 | 통계 카드 4개 (전체 상품 수·공급처 수·재고부족 수·평균 마진율) | F009 |
| 22 | 재고부족 상품 목록 알림 카드 | F010 |
| 23 | 온보딩 안내 배너 (공급처·상품 미등록 시) | F009 |

---

## 의존성 요약

```
Phase 1 (기반)
  └─ Phase 2 (공급처)
       └─ Phase 3 (상품) ← 마진 계산·재고부족 자동 전환
            ├─ Phase 4 (재고 현황)
            └─ Phase 5 (대시보드)
```

---

## 참고

- 각 Phase는 독립적으로 배포·테스트 가능
- Phase 3의 마진 자동 계산(F004)과 재고부족 자동 전환(F005)은 프론트엔드 로직으로 구현 (DB generated column 또는 클라이언트 계산)
- Recharts는 Phase 4 시작 전 설치 (`npm install recharts`)
